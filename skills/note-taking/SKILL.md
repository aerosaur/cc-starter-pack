# Note-Taking Skill

## Overview
This skill provides expert guidance on effective note-taking practices, including meeting notes, project documentation, research notes, and general knowledge capture. Focus is on clarity, searchability, and actionable information.

## When to Use This Skill
- Creating meeting notes and action items
- Documenting project information
- Capturing research and learning
- Writing daily notes and journals
- Formatting notes for clarity and readability
- Organizing knowledge bases
- Creating templates for consistent note-taking

## Core Principles of Effective Note-Taking

### 1. Clarity Over Cleverness
- Use simple, direct language
- Avoid ambiguous shorthand unless standardized
- Write for your future self who won't remember context

### 2. Action-Oriented
- Extract clear action items
- Assign owners and deadlines
- Make decisions explicit

### 3. Searchable and Scannable
- Use consistent formatting
- Include relevant keywords
- Structure with headers and lists
- Tag appropriately

### 4. Context-Rich
- Include date, attendees, purpose
- Link to related notes
- Reference source materials
- Capture the "why" not just the "what"

## Best Practices

### During Meetings
1. **Capture, don't transcribe**: Focus on key points, decisions, and actions
2. **Note ownership**: Assign action items in real-time
3. **Mark decisions clearly**: Use consistent symbols
4. **Record dissent**: When decisions aren't unanimous, note concerns

### After Meetings
1. **Send within 24 hours**: Strike while memory is fresh
2. **Highlight action items**: Make them easy to find
3. **Request review**: Ask attendees to confirm accuracy
4. **Archive properly**: File in accessible location
